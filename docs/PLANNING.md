## 1. 프로젝트 배경 및 문제 정의

### 대학 강의의 구조적 한계

대학 강의를 진행하는 교수와 수강생이 사용자이며, 해결하려는 문제는 수업 중 교수와 학생 사이의 실시간 소통 부재입니다. 대형 강의실에서 교수는 학생들의 이해도를 실시간으로 파악할 수 없습니다. "다들 이해했나요?"라고 물어도 대부분의 학생은 침묵하거나 고개를 끄덕입니다. 실제로는 이해하지 못했더라도 동료들 앞에서 모른다고 말하기 어렵기 때문입니다. 교수는 시험 성적이 나온 후에 학생들이 어디서 막혔는지 알게 되지만, 이미 진도는 나간 상태입니다. 학생 역시 질문하고 싶지만 수업 중 직접 손을 드는 것에 부담을 느끼며, 수업이 끝나면 어느 부분에서 이해가 막혔는지조차 기억하지 못합니다. 결국 교수와 학생 사이에 실시간 소통 채널이 부재하여, 수업의 질이 학생 개개인의 적극성에 의존하게 되는 구조적 문제가 있습니다.


### 문제의 핵심

Lecture Pulse의 핵심은 익명 기반의 실시간 간접 소통 채널과, 그 소통 데이터를 AI가 분석하여 교수에게 즉각적인 행동을 제안하는 것입니다. 강의자료가 연동되어 있어 학생은 수업 중에 해당 슬라이드에서 익명으로 자신의 이해 상태(이해됨, 헷갈림, 모르겠음)를 전송하고, 모르는 부분에 대해 익명 질문을 보낼 수 있습니다. 익명이기 때문에 직접 손을 들거나 말하지 않아도 되며, 내성적인 학생도 부담 없이 참여할 수 있습니다. 교수는 대시보드에서 전체 학생의 이해도 분포를 실시간으로 확인합니다. 여기에 AI 코치가 결합되어, 학생들의 혼란도가 일정 수준을 넘으면 교수에게 즉각적인 대응 방법을 자동으로 제안합니다. 예를 들어 "현재 슬라이드에서 학생들의 혼란이 집중되고 있습니다. 핵심 개념을 예시와 함께 다시 설명해보세요"와 같은 구체적인 코칭을 제공합니다. 또한 강의자료를 기반으로 AI가 퀴즈를 자동 생성하여 즉시 이해도를 확인할 수 있고, 수업 종료 후에는 AI가 전체 수업을 분석하여 다음 수업을 위한 개선안을 제시합니다.

## 2. 타겟 유저

### Primary User 1: 교강사 (Instructor)
- 대학 강의실에서 수업을 진행하는 교수, 강사, 조교
- 수업 중 학생 반응을 실시간으로 보고 싶지만, 마땅한 도구가 없어 불편함을 느끼는 사람
- 수업 후 "오늘 수업이 얼마나 잘 전달됐는지" 객관적 데이터로 확인하고 싶은 사람

### Primary User 2: 수강생 (Student)
- 강의를 수강하며 이해도가 실시간으로 변화하는 학생
- 질문은 있지만 발언 장벽 때문에 표현하지 못하는 학생
- 강의자료 요약이나 복습 지원이 필요한 학생

### 공통 전제
- 스마트폰 또는 노트북을 소지하고 있다.
- 별도 앱 설치 없이 웹 브라우저만으로 참여 가능해야 한다 (진입 장벽 최소화).
- 익명성이 보장되어야 학생의 솔직한 피드백을 끌어낼 수 있다.

## 3. 개선점

가장 큰 개선점은 교수와 학생 사이에 심리적 장벽 없는 간접 소통 채널이 생긴다는 것입니다. 기존 강의실에서 소통은 학생이 직접 손을 들고 발언하는 방식뿐이었습니다. 이 방식은 적극적인 소수의 학생에게만 유효하며, 대다수의 학생은 소통에서 배제됩니다. Lecture Pulse는 익명 실시간 피드백이라는 간접적 소통 방식을 통해, 말하지 않는 학생의 목소리까지 교수에게 전달합니다. 완전한 해결책은 아니지만, 다양한 성격의 학생과 교수 사이를 부드럽게 이어주는 매개체 역할을 합니다. 이를 통해 교수는 수업 중에 즉각적으로 수업 방식을 조정할 수 있으며, 시험 후에야 알 수 있었던 학생들의 이해 격차를 수업이 진행되는 동안 실시간으로 파악하고 대응할 수 있습니다. 수업 후에는 AI 리포트를 통해 어떤 부분에서 학생들이 어려움을 겪었는지 데이터로 확인할 수 있어, 다음 수업을 준비할 때 구체적인 근거를 갖게 됩니다. 한 번의 수업에서 끝나는 것이 아니라, 수업이 반복될수록 교수의 강의 품질이 데이터 기반으로 지속적으로 개선되는 구조입니다.

## 4. 핵심 기능 7가지

### 기능 1: 세션 관리
교수가 수업을 시작할 때 세션을 생성하면 6자리 알파벳+숫자 코드가 발급된다. 학생은 메인 페이지에서 이 코드를 입력하는 것만으로 세션에 참여할 수 있다. 앱 설치, 회원가입, 로그인이 전혀 필요 없다. 교수는 수업 종료 시 세션을 닫을 수 있으며, 닫힌 세션은 히스토리로 보관된다.

### 기능 2: 실시간 이해도 체크
학생 화면에는 3가지 이해도 신호 버튼이 있다: **이해됨 / 헷갈림 / 모르겠음**. 학생이 버튼을 누를 때마다 Supabase Realtime을 통해 교수 대시보드에 즉시 반영된다. 학생은 수업 중 언제든 신호를 바꿀 수 있으며, 각 신호의 최신 상태만 집계에 사용된다.

### 기능 3: 교수 대시보드
교수는 전용 대시보드에서 실시간 이해도 분포, 제출된 질문 목록, 활성 투표 현황을 한눈에 볼 수 있다. 도넛 차트와 막대 그래프로 시각화되어 있어 수업 중에도 빠르게 현황을 파악할 수 있다. 혼란 지수(confused + lost 비율)가 높아지면 시각적으로 강조된다.

### 기능 4: AI 질문 클러스터링
학생들이 제출한 질문이 쌓이면 교수가 "질문 묶기" 버튼을 누를 수 있다. AI(Claude Haiku)가 의미가 유사한 질문들을 자동으로 그룹화하고, API Key가 없는 환경에서는 한국어 불용어 필터링 + 토큰 오버랩 알고리즘으로 폴백한다. 클러스터로 묶인 질문은 대표 주제로 표시되어 교수가 어떤 개념이 가장 많이 막혔는지 한눈에 파악할 수 있다.

### 기능 5: 실시간 퀴즈/투표
교수가 수업 중 즉석에서 OX 퀴즈 또는 객관식 투표를 만들 수 있다. 학생에게 실시간으로 전달되고, 결과가 누적되면서 막대 그래프로 시각화된다. 교수가 투표를 닫으면 학생 화면에서도 사라진다. 수업의 이해도 확인뿐 아니라 인터랙티브한 수업 참여를 유도하는 데 활용된다.

### 기능 6: PDF 강의자료 AI 요약
교수가 강의자료 PDF를 업로드하면 Supabase Storage에 저장되고, 학생도 해당 자료를 볼 수 있다. 학생이 "AI 요약" 버튼을 누르면 Claude가 PDF를 직접 분석하여 핵심 주제, 주요 개념 bullet point, 수식/다이어그램 설명을 한국어로 요약해 준다. 한 번 생성된 요약은 DB에 캐싱되어 이후 요청 시 즉시 반환된다.

### 기능 7: 수업 후 AI 리포트
세션이 종료되면 교수는 AI가 생성한 수업 리포트를 확인할 수 있다. 리포트는 ① 전체 이해도 요약, ② 5분 단위 시간대별 이해도 추이, ③ AI가 분석한 혼란 구간과 원인, ④ 질문 클러스터 통계, ⑤ 다음 수업을 위한 AI 개선 제안 3가지를 포함한다.

## 5. AI 활용 전략

Lecture Pulse의 AI 활용은 개발 단계와 서비스 운영 단계로 나뉘며, 각 단계에서 명확한 역할 분담과 전략을 갖고 있습니다.

[개발 단계: Claude Code]
페르소나는 "프로젝트 전체를 이해하는 시니어 개발자"입니다. 단순히 코드를 생성하는 것이 아니라, 기존 코드를 먼저 읽고 프로젝트의 아키텍처와 패턴을 파악한 후에 작업을 수행하도록 합니다.
핵심 전략은 세 가지입니다. 첫째, 구현할 기능을 자연어로 한 번에 전달합니다. 예를 들어 "AI 코치 기능을 만들어줘"같이 짧게 지시하는 것이 아니라, API가 어떤 데이터를 받아서 어떤 응답을 반환해야 하는지, 대시보드에서 어떤 조건일 때 호출되는지, 트리거 조건은 무엇인지, 기존 코드 중 어떤 부분을 수정해야 하는지를 하나의 프롬프트에 모두 담아서 전달합니다. Claude Code는 이 지시를 받으면 먼저 프로젝트 코드를 읽고 기존 구조를 파악한 뒤, 새 기능을 기존 패턴에 맞춰 구현합니다. 이렇게 하면 여러 번 오가며 수정할 필요 없이 한 번의 지시로 완성도 높은 결과를 얻을 수 있고, 코드 스타일의 일관성도 유지됩니다. 둘째, 변경 범위를 명확히 제한합니다. "기존 기능은 건드리지 마"라는 제약을 항상 포함하여 의도하지 않은 사이드 이펙트를 방지합니다. 셋째, 빌드 확인을 작업의 마지막 단계로 포함시켜, 코드 생성 후 즉시 오류를 검출합니다.
제약 사항으로는, 데이터베이스 스키마 변경(ALTER TABLE 등)은 Claude Code가 직접 실행하지 않고 SQL만 출력하도록 합니다. 실제 실행은 Supabase SQL Editor에서 직접 수행하여 의도하지 않은 데이터 손상을 방지합니다.

[서비스 운영 단계: Claude Haiku API]
서비스 내부에서 Claude Haiku는 5개의 독립된 에이전트로 구성되어 각각 명확한 역할을 수행합니다. 첫 번째 에이전트는 AI 코치입니다. 페르소나는 "실시간 강의 코칭 전문가"이며, 현재 학생들의 이해도 통계와 최근 질문을 분석하여 교수에게 즉각적인 행동을 제안합니다. 입력은 이해도 통계 요약, 최근 질문 5개, 현재 슬라이드 번호, 강의자료 요약이며, 출력은 구체적 행동 제안과 퀴즈 출제 여부입니다. 클라이언트에서 혼란도 40% 이상일 때만 호출되고, 3분 쿨다운이 적용됩니다.
두 번째 에이전트는 퀴즈 생성기입니다. 페르소나는 "교육 평가 전문가"이며, 강의자료 요약 또는 특정 주제를 기반으로 OX 2문제와 객관식 1문제를 생성합니다. 출력은 반드시 JSON 배열이며, 생성된 퀴즈는 즉시 데이터베이스에 저장되어 학생들에게 실시간으로 전달됩니다.
세 번째 에이전트는 질문 분류기입니다. 페르소나는 "질문 분석 전문가"이며, 학생들의 자연어 질문을 의미 기반으로 그룹화합니다. 교수가 10개의 개별 질문 대신 2~3개의 주제로 파악할 수 있게 합니다. API 키 호출 실패 시에는 한국어 불용어를 제거한 키워드 빈도 기반 알고리즘이 자동으로 대체합니다.
네 번째 에이전트는 강의자료 요약기입니다. 페르소나는 "대학 강의자료 분석 전문가"이며, PDF에서 추출된 텍스트를 받아 핵심 개념 3~5개를 추출합니다. 요약 결과는 데이터베이스에 저장되어 코치 분석, 퀴즈 생성, 리포트에서 재활용됩니다. 학생은 시험 공부 시 이 요약을 복습 자료로 활용할 수 있습니다.
다섯 번째 에이전트는 수업 분석기입니다. 페르소나는 "교육 데이터 분석 전문가"이며, 수업 전체의 이해도 통계, 시간대별 추이, 질문 패턴, 강의자료 요약을 종합 분석하여 수업 피드백 리포트를 생성합니다. "강의자료 몇 번 슬라이드에서 혼란도가 급증했으며, 다음 수업에서 시각자료를 추가할 것을 권장합니다"와 같이 구체적인 시점과 개선안을 제시합니다.

데이터 흐름은 다음과 같습니다. 수업 전에 교수가 PDF를 업로드하면 자료 요약기가 핵심 개념을 추출하고, 이 요약은 데이터베이스에 저장됩니다. 수업 중에 학생의 이해도 신호와 질문이 실시간으로 수집되면, 질문 분류기가 유사 질문을 그룹화하고, AI 코치가 혼란도를 감지하여 교수에게 제안합니다. 교수가 퀴즈를 요청하면 퀴즈 생성기가 강의자료 기반으로 즉시 생성합니다. 수업 후에는 수업 분석기가 모든 데이터를 종합하여 다음 수업을 위한 리포트를 생성합니다. 모든 에이전트는 동일한 강의자료 요약 데이터를 공유하여 중복 분석을 방지하고, 수업 내용과 연결된 일관된 분석을 제공합니다.

### AI 폴백 설계

**ANTHROPIC_API_KEY가 없는 환경**에서도 핵심 기능이 동작해야 한다:

- **클러스터링 폴백**: 한국어/영어 불용어 제거 → 토큰 추출 → 부분 일치(partialMatch) 기반 오버랩 점수 계산 → 0.2 이상이면 같은 클러스터. 단독 질문은 클러스터에서 제외.
- **요약 폴백**: 요약 버튼 클릭 시 "API 키가 설정되지 않았습니다" 메시지 표시 (UX 저하 최소화).
- **리포트 폴백**: AI 요약 없이 통계 데이터만으로 구성된 리포트 반환.

## 6. 기술 아키텍처 개요

```
학생 브라우저 ──── Next.js App Router ──── Supabase PostgreSQL
     ↕ Realtime                ↕ API Routes         ↕ Storage (PDF)
교수 브라우저      Anthropic Claude API    Supabase Realtime
```

- **프론트엔드**: Next.js 16 (App Router), React 19, Tailwind CSS 4
- **백엔드**: Next.js Route Handlers (서버리스)
- **DB**: Supabase PostgreSQL + Realtime WebSocket
- **파일 스토리지**: Supabase Storage (PDF)
- **AI**: Anthropic Claude API (Haiku)
- **배포**: Vercel (Next.js 최적화 환경)
